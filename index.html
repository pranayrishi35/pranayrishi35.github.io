Project Overview

Project Suraksha QR was initiated to address the escalating crisis of trust in India's digital payment ecosystem. The mission is to re-architect the security paradigm, shifting the burden of fraud detection from the vulnerable end-user to an intelligent, empathetic, and resilient technological framework.

Key Innovations & Features

  • Real-time Transaction Risk Engine (RTRE): An AI model that analyzes transactions for "scam signatures" beyond simple anomaly detection.
  • Proactive Trust Signals: A QR Verification & Rating system provides users with an immediate trust signal before they transact.
  • Time-Lock Protocol: High-risk transactions are automatically placed on a temporary hold, giving users a crucial window to reconsider and preventing coerced payments.
  • Hardened Three-Factor Verification (3FV): For the most critical transactions, a robust, multi-step verification process is enforced, creating an exceptionally strong defense.

Prototype Architecture

The project is a self-contained prototype demonstrating the complete security flow. The backend is a simulated secure API written in JavaScript, encapsulating the risk engine and security layers (HMAC signing, input validation, rate limiting) to mirror a production environment without exposing core logic on the client-side.

Strategic Vision

With the prototype complete, the project is primed for a "Security-as-a-Service" (SaaS) business model, aiming to secure a pilot program with a major banking partner to build a long-term competitive moat through the network effect of its security data.
